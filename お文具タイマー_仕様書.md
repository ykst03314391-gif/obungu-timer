## 1. 概要

子ども向けのスケジュール通知・タイマーアプリ。設定した時刻になるとキャラクターとBGM・音声で通知する。親が設定画面でイベントを登録し、子どもが表示画面で確認する2画面構成。

---

## 2. 技術スタック

| 項目 | 内容 |
|---|---|
| 構成 | シングルHTMLファイル（index.html） |
| スタイル | インラインCSS |
| ロジック | バニラJS（フレームワークなし） |
| 音声合成 | Web Speech API |
| 音声再生 | Web Audio API（fetch + decodeAudioData） |
| データ永続化 | LocalStorage |
| ホスティング | GitHub Pages |

## 3. 画面構成

### 3-1. 設定画面（親用）

アプリ起動時は表示画面が表示される。左上の ⚙️ ボタンで設定画面へ遷移。

| カード | 内容 |
|---|---|
| イベント追加フォーム | イベントのCRUD操作 |
| イベント一覧 | 登録済みイベントの表示・編集・削除 |
| その他 | 通知フラグリセットボタン |
| エクスポート / インポート | CSVによる設定の保存・復元 |

### 3-2. 表示画面（子ども用）

| 要素 | 内容 |
|---|---|
| 年月日 | 現在の日付を上部に表示 |
| 現在時刻 | 大きく中央に表示（HH:MM） |
| キャラクター | フェーズに応じたGIFアニメ |
| 次のイベント名 | 直近イベントの名称 |
| 設定時刻 | 「設定時刻：HH:MM」形式 |
| カウントダウン | 「あと HH:MM:SS」（1時間以内のみ表示） |
| 単発タイマーバー | タイマー動作中のみ表示 |
| アクションボタン | フェーズに応じて「準備完了 ✅」「できた！✅」「早期完了 ✅」を表示 |

## 4. イベント管理

### 4-1. イベントデータ構造

```
{
  id:             string,   // タイムスタンプベースのID
  name:           string,   // イベント名（最大20文字）
  time:           string,   // 時刻（"HH:MM"形式）
  days:           number[], // 繰り返し曜日（0=日〜6=土）空配列=毎日
  alertBefore:    number,   // 事前通知（分前）: 5/10/15/20/25/30
  preMessage:     string,   // 事前通知メッセージ（最大50文字）
  onTimeMessage:  string,   // 時刻ちょうどメッセージ（最大50文字）
  doneMessage:    string,   // 完了メッセージ（最大50文字）
  enabled:        boolean,  // 有効/無効
}
```

### 4-2. LocalStorageキー

| キー | 内容 | リセットタイミング |
|---|---|---|
| `obungu_timer_events` | イベント設定（永続） | 手動のみ |
| `obungu_fired_today` | 通知済みフラグ | 日付変わり時 |
| `obungu_done_today` | 完了済みフラグ | 日付変わり時 |
| `obungu_chara_index` | キャラインデックス | なし（循環） |

### 4-3. CSVフォーマット

```
イベント名,時刻,繰り返し,事前通知(分),事前通知メッセージ,時刻メッセージ,完了メッセージ
おふろ,19:00,日月火水木金土,10,そろそろじゅんびしようね,じかんだよ！,えらい！
```

繰り返しが毎日の場合は「毎日」または空文字。

## 5. 通知フェーズ

| フェーズ | 背景色 | 発生条件 | キャラ |
|---|---|---|---|
| normal | オレンジ〜クリーム | 通常 | 現在キャラのnormal GIF |
| alert | 黄色 | 設定時刻のX分前〜設定時刻 | 現在キャラのalert GIF |
| ontime | 緑〜水色 | 準備完了済みで設定時刻到来 | 現在キャラのontime GIF |
| panic | ピンク〜赤 | 未準備で設定時刻到来 | ゼリーさん固定（シェイクアニメ） |

### 5-1. 通知タイミング

| イベント | 発火条件 | 発火回数 |
|---|---|---|
| 事前通知（alert） | 設定時刻のalertBefore分前〜設定時刻の間 | 1回/日 |
| 時刻通知（ontime） | 設定時刻±60秒のウィンドウ | 1回/日 |
| panic BGM自動停止 | 設定時刻の5分後 | 1回/日 |

## 6. アクションボタン

| ボタン | 表示条件 | 動作 |
|---|---|---|
| 準備完了 ✅ | alertフェーズ中、通知ポップアップ内 | BGM停止・junbi-overlay表示・yeah.mp3再生・normalフェーズへ |
| できた！✅ | ontime/panicフェーズ中 | イベント完了マーク・BGM停止・clear.m4a再生・やったね！画面表示 |
| 早期完了 ✅ | 準備完了済み・設定時刻前・normalフェーズ | イベント完了マーク・BGM停止・clear.m4a再生・やったね！画面表示 |

## 8. やったね！画面

- 現在キャラのalert GIF + コンフェッティアニメーション表示
- 完了メッセージ（イベントの`doneMessage`、または単発タイマーのイベント名）
- 「とじる」ボタンで閉じ、次のキャラへローテーション

## 9. キャラクター

- 複数キャラが定義されており、アプリリロードごとに循環
- 各キャラに `normal` / `alert` / `panic` の3種GIFが存在（Base64埋め込み）
- panicフェーズは全キャラ共通で「ゼリーさん」を表示

## 10. 音声

### 10-1. BGM（Web Audio API・ループ再生）

| ファイル | 用途 | 音量 |
|---|---|---|
| `sounds/kb.m4a` | 事前通知BGM（alertフェーズ） | 0.12 |
| `sounds/dash.mp3` | 時刻到来BGM（ontime/panicフェーズ） | 0.12 |

### 10-2. 効果音（Web Audio API・1回再生）

| ファイル | 用途 |
|---|---|
| `sounds/yeah.mp3` | 準備完了ボタン押下時 |
| `sounds/clear.m4a` | できた！ボタン押下時（イベント・単発タイマー共通） |

### 10-3. 音声合成（Web Speech API）

| フェーズ | rate | pitch | volume |
|---|---|---|---|
| normal | 0.88 | 1.05 | 0.90 |
| alert | 0.92 | 1.08 | 0.95 |
| panic | 0.98 | 1.05 | 1.00 |

- 優先音声：Microsoft Haruka（Japanese Japan）
- `soundsフォルダ` の音声ファイルは `fetch()` で読み込むため、GitHub Pages等のHTTPサーバー必須

## 11. 単発タイマー

表示画面右上の ⏱ ボタンから起動。スケジュールイベントとは独立した即時カウントダウン機能。

### 11-1. 設定

| 項目 | 内容 |
|---|---|
| 分数入力 | 1〜99分（2桁制限）、デフォルト空白 |
| プリセット | 5分 / 15分 / 30分ボタン |
| イベント名 | 任意入力（最大40文字）、残り時間の上に表示 |

### 11-2. 動作フロー

```
スタート押下
  → オーバーレイを閉じ、本体画面にタイマーバーを表示
  → MM:SS形式でカウントダウン

  ┌ できた！✅ 押下（時間内）
  │  → clear.m4a再生・タイマーバー非表示・やったね！画面表示
  │
  ├ やめる 押下（時間内）
  │  → タイマー停止・タイマーバー非表示
  │
  └ 時間切れ（0:00）
     → ラベルが「じかんがきたよ！」に変わる
     → panicフェーズ移行・dash.mp3 BGM再生
     → 「やめる」ボタンのみ残る
        → 押下でBGM停止・normalフェーズ復帰

終了時（いずれの場合も）: 分数・イベント名入力欄をクリア
```

---

## 12. カウントダウン表示ルール

- 直近の未完了イベントが **1時間以内** の場合のみ表示
- `あと HH:MM:SS` 形式（正値）
- 設定時刻超過中（panicウィンドウ内）は `+MM:SS` 形式（負値）
- 1時間超先または当日イベントなしは「よていなし」表示
